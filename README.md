# portal-main-resource

一个用于归档和发布多个独立 HTML 页面的通用仓库。本仓库不针对任何特定网站或域名，仅作为静态页面的集中存放与版本管理用途。

## 项目简介

portal-main-resource 旨在提供一个简洁、通用的 HTML 页面归档空间。无论是临时页面、功能演示、文档说明，还是其他独立的网页资源，均可通过此仓库进行统一管理和发布。每个页面独立存在，互不依赖，便于快速部署与分享。

## 目录说明

```
portal-main-resource/
├── pages/               # 存放所有独立 HTML 页面
│   ├── example1.html    # 示例页面（可删除或替换）
│   └── example2.html
├── assets/              # 页面引用的公共资源（如 CSS、JS、图片）
├── index.html           # 仓库首页（可选，用于展示页面列表或导航）
└── README.md            # 本文件
```

- **pages/**：所有归档的 HTML 页面应放置于此目录，每个文件对应一个独立页面。
- **assets/**：如需共享样式、脚本或图片，可放入此目录，页面通过相对路径引用。
- **index.html**：可自定义为页面索引或导航页，便于浏览仓库内容。

## 页面归档说明

- 所有页面均为独立、自包含的 HTML 文件，允许引用 `assets/` 目录下的公共资源。
- 页面命名建议使用有意义的英文或拼音，避免特殊字符。
- 页面内不应包含任何动态后端逻辑，仅限静态内容。
- 如需更新页面，直接修改对应文件并提交即可；版本历史由 Git 自动记录。

## 维护说明

- 欢迎提交新的 HTML 页面，请将其放入 `pages/` 目录，并确保文件格式正确、无外部依赖（除 `assets/` 外）。
- 如需修改现有页面，请遵循常规 Pull Request 流程。
- 本仓库不承担页面内容的审核责任，但建议保持内容合法、合规。
- 如发现页面损坏或链接错误，欢迎提交 Issue 反馈。

## 如何开始

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加你的 HTML 文件。
3. 可选：编辑 `index.html` 更新页面列表。
4. 提交并推送更改。
